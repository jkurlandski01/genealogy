SVN Examples/Tasks

JERRY: Red bar at top needs to point to this page's subtopic

This page shows you how to link individual SVN commands together in order to accomplish a particular task.

The command line can be a little intimidating at first, but I find it becomes less so when you see the commands not in isolation but as part of a series of steps with a single aim in mind. It's kind of like the way individual words don't make sense until strung together into a coherent sentence.

Listing X: Accidentally adding directories, and reverting the mistake

Jerrys-MacBook-Pro:jkurlandski jkurlandski$ svn status
?       jkurlandskiWebsite/Index/Professional/Misc/svnExamples.html
?       jkurlandskiWebsite/Index/Professional/toDo.txt

Jerrys-MacBook-Pro:jkurlandski jkurlandski$ svn commit -m "New svnExamples page."
Jerrys-MacBook-Pro:jkurlandski jkurlandski$ svn status
?       jkurlandskiWebsite/Index/Professional/Misc/svnExamples.html
?       jkurlandskiWebsite/Index/Professional/toDo.txt

(Commit failed because I still haven't added the new files to the repository.)

Jerrys-MacBook-Pro:jkurlandski jkurlandski$ svn add *
svn: warning: W150002: '/Users/jkurlandski/Documents/Programming/WebDevelopment/Code/jkurlandski/CodeExamples' is already under version control
A         RemoteSystemsTempFiles
A         RemoteSystemsTempFiles/.project
svn: warning: W150002: '/Users/jkurlandski/Documents/Programming/WebDevelopment/Code/jkurlandski/branches' is already under version control
svn: E200009: Could not add all targets because some targets are already versioned
svn: E200009: Illegal target for the requested operation

(The intention is to add the new files--but as you'll see, the approach taken is a mistake.)

Jerrys-MacBook-Pro:jkurlandski jkurlandski$ svn status
A       RemoteSystemsTempFiles
A       RemoteSystemsTempFiles/.project
?       jkurlandskiWebsite/Index/Professional/Misc/svnExamples.html
?       jkurlandskiWebsite/Index/Professional/toDo.txt


(We see the command added ignored directories, and failed to add new files in versioned directories.)

(Let's revert the add.)

Jerrys-MacBook-Pro:jkurlandski jkurlandski$ svn revert RemoteSystemsTempFiles
svn: E155038: Try 'svn revert --depth infinity' instead?
svn: E155038: Can't revert '/Users/jkurlandski/Documents/Programming/WebDevelopment/Code/jkurlandski/RemoteSystemsTempFiles' without reverting children

(Must first revert the child, then the parent.)

Jerrys-MacBook-Pro:jkurlandski jkurlandski$ svn revert RemoteSystemsTempFiles/.project
Reverted 'RemoteSystemsTempFiles/.project'
Jerrys-MacBook-Pro:jkurlandski jkurlandski$ svn revert RemoteSystemsTempFiles
Reverted 'RemoteSystemsTempFiles'
Jerrys-MacBook-Pro:jkurlandski jkurlandski$ svn status
?       jkurlandskiWebsite/Index/Professional/Misc/svnExamples.html
?       jkurlandskiWebsite/Index/Professional/toDo.txt

(Reversion successful, but we still haven't added the new files. We could do each one separately, e.g. with "svn add jkurlandskiWebsite/Index/Professional/toDo.txt"; we could do both together with "svn add * --force," but this would again add the ignored files we've just reverted; instead we'll do them both with a Unix shell command.)

Jerrys-MacBook-Pro:jkurlandski jkurlandski$ for i in $(svn status | grep \? | awk '{print $2}'); do svn add $i; done
A         jkurlandskiWebsite/Index/Professional/Misc/svnExamples.html
A         jkurlandskiWebsite/Index/Professional/toDo.txt

(Check status.)

Jerrys-MacBook-Pro:jkurlandski jkurlandski$ svn status
A       jkurlandskiWebsite/Index/Professional/Misc/svnExamples.html
A       jkurlandskiWebsite/Index/Professional/toDo.txt
