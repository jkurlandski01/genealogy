  • $ git diff --staged

    • (Same as "git diff --cached".) Shows what you've staged that will go into your next commit. This is useful because "git diff" by itself will return nothing if you've already staged all your changes--that is, it only displays changes that are still unstaged.

  • $ git difftool --tool-help

    • Shows tools on your system that are available for viewing the diffs.

  • $ git commit

    • Note that all the "git commit" commands operate only on items that have already been staged.

  • $ git commit

    • This command launches an editor of your choice where you can see what you're committing and insert a message to go with the commit.

  • $ git commit -m ""

    • Commits with your message, not opening up an editor.

  • $ git commit -a -m ""

    • The "-a" option lets you skip the staging area altogether, and commit directory to the repository. Note that it applies only to files currently being tracked--that is, if the file is new, you must first add it to begin tracking on the file; only then can you commit it to the repository with the "commit -a " option.

  • $ git rm item

    • Removes an item from git tracking, and also removes it from your file system.

  • $ git rm --cached item

    • Removes an item from git tracking, but retains it in your file system.

  • $ git mv

    • Renames an item, retaining file history. But note that git is capable of recognizing file renames performed via the operating system. In other words, use of the mv command is the same as the following three-step process.

    mv  
git rm 
git add

    File-glob patterns

  • $ git rm log/\*.log

    • Removes all files ending in .log from the log directory.

    Undoing Actions

  • $ git commit -amend

    • (Assuming you've made no changes since your last commit; e.g. you run it just after a commit.) Launches your text editor with the message of your last commit, allowing you to edit the message.

    (Assuming you've committed, realized you forgot to include a file, and have since added the file to staging.) Launches your text editor with the message of your last commit, allowing you to edit the message--and includes the missing file in the same commit as the previously-committed files.

  • $ git reset HEAD <item>

    • Removes an item added to staging--e.g. so that you can commit something else added at the same time.

  • $ git checkout -- <item>

    • Reverts changes to the item. Dangerous in the sense that there is no way to recover your changes to the file.
