Sorting: map keys, Arrays.sort(), Collections.sort()

and TreeMap

This is not a complicated Java code on this page, but sometimes I forget exactly how to use the sorting techniques outlined here. I wrote this page to serve as a quick reference for myself or others.

There are a couple of ways to sort on the keys of a map. You could: (1) use Java's TreeMap, or (2) get the key set and sort that. If your requirements are as simple as this, this page will help you out. If your requirements are more complicated, you might want to explore Guava's Table interface and its various implementations. I have a few pages dedicated to making use of Guave to implement a table data structure that can be sorted on any of its various columns. See the links at right.

Using TreeMap

The listing below shows you how to use a TreeMap, which is a specialized Java map that returns the keys in sorted order.

Listing 1: Using TreeMap to sort the index of an in-memory table

TreeMap<String, Double> map1 = new TreeMap<String, Double>();
map1.put("one", new Double(1.0));
map1.put("two", new Double(2.0));
map1.put("three", new Double(3.0));
        
// Convert to list for convenience.
List<String> keyList = Lists.newArrayList(map1.keySet());

Here I initialize the map in the traditional Java 1.6 way, but you if you have Guava you can do it more simply with Maps.newTreeMap(). Also, you can work with the map if you want, but sometimes you want those sorted rows as a List of strings.
