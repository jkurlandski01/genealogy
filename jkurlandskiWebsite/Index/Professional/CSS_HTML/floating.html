In CSS / HTML

The code

Here are the CSS definitions for the floating table above as well as the caption that you see beneath it.

Listing 1: CSS definition for floating table and its caption

floatedtable {
	float: right;
	padding: 10px;
	caption-side:bottom;
}
figcaption p { 
	line-height: 1;
	font-size: 10pt;
	font-family: 'FontAwesome';
}

Here's how you use these definitions in the html. The only tricky thing is that in the html the table comes before the text that you want to pair it with. What is really floating, then, is the text--not the table--in the sense that the text "floats" upward until it's level with the table.

Listing 2: Using the floating table styles in HTML

<div class="floatedtable">
	<table border="1">
		<tr>
			<th>Title</th>
			<th>Director</th>
			<th>Country</th>
		</tr>
		<tr>
			<td>Seventh Seal</td>
			<td>Ingmar Bergman</td>
			<td>Sweden</td>
		</tr>
		<tr>
			<td>Seven Samari</td>
			<td>Akira Kurosawa</td>
			<td>Japan</td>
		</tr>
		<tr>
			<td>Seven</td>
			<td>Andy Sidaris</td>
			<td>United States</td>
		</tr>
	</table> 
	<div class="figcaption">
		<p>A floating table</p>
	</div>
</div>

If your table has too many columns, or the contents of the columns are too long, you'll want to use a traditional table. An example is the very first table at the top of this page. The HTML for that looks like this.

Listing 3: HTML for a traditional table

<table border="1" cellpadding="4px">
	<tr>
		<th>Title</th>
		<th>Director</th>
		<th>Country</th>
	</tr>
	<tr>
		<td>Seventh Seal</td>
		<td>Ingmar Bergman</td>
		<td>Sweden</td>
	</tr>
	<tr>
		<td>Seven Samari</td>
		<td>Akira Kurosawa</td>
		<td>Japan</td>
	</tr>
	<tr>
		<td>Seven</td>
		<td>Andy Sidaris</td>
		<td>United States</td>
	</tr>
</table> 
<div class="figcaption">
	<p>This table is styled normally.</p>
</div>

Notice that the styling above calls for a little bit of cell padding. I think that the table looks a tad nicer this way, and since the table isn't floating you probably have the room for it.

But what if you DON't want to float?

If you don't want text alongside your table, simply don't float the table--use the normal table html given above. But suppose you have two small tables, or two thumbnail images, and you want them to appear side-by-side? In this case you would float them both. The problem is that, if you float the two tables or images, then any text that follows them will be placed next to them.

Here is an illustration of the problem I'm describing.

A floating table

Here's some text I DON'T want to appear alongside the tables.

To get around this, you need to clear the floating style. A simple way to do this is to use a CSS style for the paragraph which you don't want to float.

Here's the very simple CSS definition:

Listing 4: CSS for a non-floating paragraph

.nonFloatingPar	p	{
	clear: both;
}

And here's the HTML that makes use of the definition:

Listing 5: Using the non-floating paragraph style in HTML

<div class="nonFloatingPar">
	<p>Here's some text I DON'T want to appear alongside the tables.</p>
</div>

Sometimes you want to stop all floating of any elements below a certain point on the page. This snippet makes use of the nonFloatingPar style with an empty paragraph.

Listing 6: Using the non-floating paragraph style to stop floating from this point on the page, down.

<div class="nonFloatingPar">
	<p></p>
</div>

Displaying CSS and HTML code on a webpage

JERRY: Need link to my page. Probably go into too much detail here.

By the way, it's tricky getting CSS and HTML to appear on a web page--the browser tries to render it the same way it renders the rest of your page. Css can be enclosed in <code> or <pre> tags. You're supposed to be able to do the same with html, but in my experience you need the special tags AND the html code itself has to be converted: the "<" symbol must be replaced with "&lt;" (think "less than"); the ">" symbol with "&gt;" (think "greater than"); and "&" with "&amp;". The conversion is unpleasant business. Eclipse offers a tool that will do this for you--see a separate page in this section of Close to the Machine.
