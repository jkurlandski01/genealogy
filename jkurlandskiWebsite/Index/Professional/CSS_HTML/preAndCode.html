Using <pre> and <code>

to display code


Sometimes website developers want to share a technique that they've discovered or found useful. Sometimes they simply want to set a few lines apart from the rest of the text on a web page. The <pre> and <code> tags are useful for this purpose.

If you're in a hurry (or if you don't like to read): Jump to the last two listings on this page.

Converting reserved HTML characters

When trying to display HTML and other programming code on a web page, one of the first obstacles that has to be overcome is the fact that certain characters, particularly angle brackets, are reserved. This means that browsers interpret these characters differently from other characters, and will not display them on the web page. To get these characters to display correctly, you have to type into your HTML not the character itself, but its entity equivalent.

I explain HTML entities in greater detail on a separate page—see the link in the sidebar. On this page, we're concerned exclusively with those characters that you find in computer code. What you need to do to display HTML and most computer code on your web page is the following:

  • replace the "<" symbol with "&lt;" (think "less than");
  • replace the ">" symbol with "&gt;" (think "greater than");
  • replace the "&" symbol (the ampersand) with "&amp;".

Introducing the <pre> and <code> tags

Use the <pre> tag to set your code off from the regular formatting of the page. Compare the next few lines of CSS with the text of this paragraph.

Listing 1: Your browser's default settings for <pre>.

someTag {
    margin: 0px;
    //3456789012
}

Each browser has its own default version of this tag. Since <pre> is not defined in my CSS stylesheet, the snippet of code above illustrates your browser's default setting for rendering text surrounded by the pre tag. Most browswers will assign a monospace type to text wrapped by this tag.

The next snippet contains the exactly the same text, but wrapped in the <code> tag instead.

Listing 2: Your browser's default settings for <code>.


someTag { margin: 0px; //3456789012 }

You see that the <code> tag also assigns a monospace font, but it seems less useful, if only because browsers typically ignore the contents' newlines and collapse the whitespace. If whitespace is important in your display of code (and usually it is to the reader, if not the compiler), then you need <pre>. The <code> tag can be used for inline elements, like the following: public static int HUNDRED = 100;

Some will say that you should present blocks of code with both <pre> and <code>. But, so far, for blocks of HTML or other code, I am unpersuaded; incuding it just seems like extra work to me.

A pretty good pre tag styling

After some experimenting I finally found a style for <pre> that suits my needs. My first decision was whether to define the pre tag's styling, or to create my own class. Had I decided on the former, then all usages of the tag would have been determined by that style, making it impossible for me to use the default stylings. In other words, Listing 1 on this page would have been styled in exactly the same wasy as Listing 3. I decided I did not want that. So I created a new class, calling it "preCode," because its general behavior is like that of <pre>, and because its primary usage is to display code like HTML.

Listing 3: A pretty good styling for displaying code with <pre>.

pre.preCode
{
    font-family: "Courier New" Courier monospace;
    font-size: 0.80em;
    background: #E8E8E8;
    overflow: auto;
    padding: 10px;
    // a long commented out line that displays the scrolling feature if it is ever needed one: two three four five six seven eight nine ten  
}

In the CSS snippet above, we first state that we're creating the preCode class. Then we say we want to use Courier or a similar monospace type, and we define the font size. Then we give the text a slightly tinted background. Then we handle lines that are too long with scrolling. Finally we want a little whitespace between the text and the edges of the gray box—otherwise the text abuts the edges and everything looks too crowded. The last line is not actually part of the CSS style; I inserted it simply in order to display how the overflow setting makes scrolling behave.

Here's how to use the preCode class I've just defined.

Listing 4: How to use the preCode style in HTML.

<pre class="preCode">
...
</pre>

The ellipses represent whatever you want the viewer to see. Don't forget that whatever takes the place of these ellipses must have its angle brackets "escaped," as described in the separate section above.

By the way, <pre> is so literal that you can't indent it. The next listing is the same as the above, but with the tags indented, as HTML normally is. Note the extra space at the bottom of the tinted window. That space is due to the whitespace between the left margin and the closing </pre> tag.

Listing 5: The same as Listing 4, but with the tags indented.

     <pre class="preCode">
...
     </pre>

The problem is, if you don't indent it, the code becomes a little harder to read; therefore, I find my HTML is clearest if I leave blank lines before and after my unindented <pre> blocks.

A styling for a pre block with captions

One thing I don't like about the pre blocks you've been looking at thus far is the spacing between the caption and the pre block window. Let's create a new style for the case when you want to assign a caption to this block (which for me is most of the time).

Listing 6: A style for a preCode block with a caption, and the style of its caption.

.precaption {
    line-height: 1;
    font-size: 10pt;
    font-family: 'FontAwesome';
    font-weight: 600;
    margin-bottom: 0px;
    padding-bottom: 0px;
}

pre.preCodeWithCaption
{
    margin-top: 0px;
    font-family: "Courier New" Courier monospace;
    font-size: 0.80em;
    background: #E8E8E8;
    overflow: auto;
    padding: 10px;
    // a long commented out line that displays the scrolling feature if it is ever needed one: two three four five six seven eight nine ten  
}

The definition of the caption gives it some desirable characteristics, like a slightly smaller, slightly darker font. Note the bottom margin and padding, which are set to 0. These go hand-in-hand with the top margin of the preCodeWithCaption class. These values combine so that the caption appears to just touch the tinted block. Apart from the margin-top setting, the preCodeWithCaption class is identical to preCode.

And here's how to use it in HTML.

Listing 7: Using preCodeWithCaption, and giving your code block a caption.

<p class="precaption">Your caption here</p>

<pre class="preCodeWithCaption">
...
</pre>
