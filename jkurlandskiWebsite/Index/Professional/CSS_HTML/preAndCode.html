Using <pre> and <code>

to display code

JERRY: if you use the guy with the detailed example extensively, then reference him; add to sidebar

Sometimes website developers want to share a technique that they've discovered or found useful. Sometimes they simply want to set a few lines apart from the rest of the text on a web page. The <pre> and <code> tags are useful for this purpose.

Converting, or "Escaping," special characters in HTML, Java, and other languages

JERRY: Use the term entities.

It's a little tricky getting HTML to appear on a web page--the browser tries to render it the same way it renders the rest of your page. For example, when you want to talk about the <p> tag, you can't type in the angle brackets on either side of the "p". If you used the angle brackets, browsers would try to interpret them literally.

To get around this, you have to replace the < and > symbols on either side of the "p". And in other cases you'll also have to replace the ampersand charcter. In other words:

  • the "<" symbol must be replaced with "&lt;" (think "less than");
  • the ">" symbol must be replaced with with "&gt;" (think "greater than");
  • the "&" symbol (the apersand) must be replaced with "&amp;".

To make this perfectly clear, in order to display this on my screen--<p>--I must type in this: &lt;p&gt; And in order to display this on my screen--&--I have to type in this: &amp;

As you can imagine, this conversion is unpleasant business. Eclipse offers a command that will do this for you automatically.

It's not just  HTML that has to be transformed in this way. Java's types--as in List<String>--as well as the "&" symbol in C and C++--will all have to be converted in this manner. So a tool that does this for you automatically is a very useful tool indeed.

			
		
Introducing the <pre> and <code> tags

		
Use the <pre> tag to set your code off from the regular formatting of the page. Compare the next few lines of CSS with the text of this paragraph.

		
someTag {
    margin: 0px;
    //3456789012
}
		

		
Each browser has its own default version of this tag. Since <pre> is not defined in my stylesheet, however you see the snippet of code above illustrates how your browser renders text by default. Most browswers will assign a monospace type to text wrapped by this tag.

		
The next snippet contains the same lines, but wrapped in the <code> tag instead.

someTag {
    margin: 0px;
    //3456789012
}
		
		

		
		
You see that the <code> tag also assigns a monospace font, but it seems less useful, if only because browsers typically ignore the contents' newlines and collapse the whitespace. If whitespace is important in your display of code (and usually it is to the reader, if not the compiler), then you need <pre>. The <code> tag can be used for inline elements, like the following: public static int HUNDRED = 100; 

		
Some will say that you should present blocks of code with both <pre> and <code>. But I don't see the point of the code tag; incuding it just seems like extra work to me.

		
		
A few experiments

		
Let's play around with these tags, setting the style sheet first and then inserting the corresponding HTML in the tag just defined.

		
pre.simple
{
    font-family: "Courier New" Courier monospace;
    background: #E8E8E8;
    overflow: auto;
    padding-top: 10px;
    padding-left: 5px;
    // a long commented out line that displays the scrolling feature if it is ever needed one two three four five six seven eight nine ten
}
		

		
JERRY: I think I need the padding-top because of something in my stylesheet. Maybe a bottom border on the p tag?

		
This is a pretty basic definition that probably suits most needs. First we define the typeface to be monospace, just in case the browser isn't that way inclined. Then we give the text a slight greyish background. Then we handle lines that are too long with scrolling. Finally we fix an issue that leaves more space at the top of the text than at the bottom. And the padding-left setting gives the text a little bit of breathing room from the margin on that side.

		
On the HTML side, simply wrap your text with the following. (And don't forget to "escape" the angle brackets as described above.)

		
<pre class="simple">
...
</pre>
