Using <pre> and <code>

to display code

JERRY: if you use the guy with the detailed example extensively, then reference him; add to sidebar

Sometimes website developers want to share a technique that they've discovered or found useful. Sometimes they simply want to set a few lines apart from the rest of the text on a web page. The <pre> and <code> tags are useful for this purpose.

Escaping Code

It's a little tricky getting css and html to appear on a web page--the browser tries to render it the same way it renders the rest of your page. In other words, when you want to talk about the <p> tag, you have to tell the browser that it should not interpret the tag literally--that is, you want to "escape" the < and > symbols on either side of the "p". The special HTML characters have to be converted: the "<" symbol must be replaced with "&lt;" (think "less than"); the ">" symbol with "&gt;" (think "greater than"); and "&" with "&amp;". The conversion is unpleasant business.

Eclipse offers a command that will do this for you automatically.

Introducing the <pre> and <code> tags

Use the <pre> tag to set your code off from the regular formatting of the page. Compare the next few lines of CSS with the text of this paragraph.

someTag {
    margin: 0px;
    //3456789012
}

Since <pre> is not defined in my stylesheet, each browser is free to render its own default version of this tag. Most browswers will assign a monospace type to text wrapped by this tag.

The next few lines are wrapped in the <code> tag.

someTag { margin: 0px; //3456789012 }

You see that the <code> tag also assigns a monospace font, but it seems less useful, if only because browsers ignore the contents' newlines and collapses the whitespace. If whitespace is important in your display of code (and usually it is to the reader, if not the compiler), then you need <pre>. The <code> tag can be used for inline elements, like the following: public static int HUNDRED = 100;

Some will say that you should present blocks of code with both <pre> and <code>. But I don't see the point of the code tag; it's just seems like extra work to me.

A few experiments

Let's play around with these tags, setting the style sheet first and then inserting the corresponding HTML in the tag just defined.

pre.simple
{
    font-family: "Courier New" Courier monospace;
    background: #E8E8E8;
    overflow: auto;
    padding-top: 10px;
    padding-left: 5px;
    // a long commented out line that displays the scrolling feature if it is ever needed one two three four five six seven eight nine ten
}

This is a pretty basic definition that probably suits most needs. First we define the typeface to be monospace, just in case the browser isn't that way inclined. Then we give the text a slight greyish background. Then we handle lines that are too long with scrolling. Finally we fix an issue that, without the padding, has the first line of text abut the top of the grey area, while leaving a line of empty space below the last line. And the padding-left setting gives the text a little bit of breathing room from the margin on that side.

On the HTML side, simply wrap your text with the following. (And don't forget to "escape" the angle brackets as described above.)

<pre class="simple">
...
</pre>

That's the last time I'll provide the HTML code. From now on it'll be just CSS--don't forget to modify the class value accordingly.
