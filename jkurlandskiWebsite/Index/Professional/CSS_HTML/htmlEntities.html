HTML Entities and Reserved Characters


This page discusses HTML entities, which are special codes that tell browsers how to display characters that you can't ordinarily type, for example the ¢ sign. Entities are also useful for reserved characters such as the angle brackets and the ampersand.

HTML reserved characters

It's a little tricky getting HTML to appear on a web page—the browser tries to render it the same way it renders the rest of your page. For example, when you want to talk about the <p> tag, you can't type in the angle brackets on either side of the "p". If you used the angle brackets, browsers would try to interpret them literally.

To get around this, you have to replace reserved characters like the < and > symbols, as well as the ampersand character, with what are called HTML character entities. In other words:

  • the "<" symbol must be replaced with "&lt;" (think "less than");
  • the ">" symbol must be replaced with "&gt;" (think "greater than");
  • the "&" symbol (the ampersand) must be replaced with "&amp;".

To make this perfectly clear, in order to display this on my screen—<p>—I must type in this: &lt;p&gt; And in order to display this on my screen—&—I have to type in this: &amp;

It's not just HTML that has to be transformed in this way. Java's generic types--as in List<String>—as well as the "&" symbol in C and C++—will all have to be converted in this manner.

As you can imagine, this conversion is unpleasant business. Aptana, a plug-in for the Eclipse IDE, offers a command that will do this for you automatically. See my page on Aptana and its tools.

HTML entities

The cent sign, ¢, is just one example of a plethora of special characters that are not on the average computer keyboard, but which you might well want to put on your web page. For each entity, there are two representations, whose syntax is either &entity_name; or &#entity_number;, respectively. You see how the entities for the angle brackets and the ampersand given in the section above fall into the first pattern.

There are some symbols for which no entity name pattern exists—they have only an entity number pattern. Also, supposedly there is better support in various browsers for the entity number patterns than for the entity name patterns. However, for the special characters that I use on my own web pages, I have never found an entity name pattern not to display properly in the most popular browsers.

The lists below are meant to serve as a one-stop place where I can find the HTML entities for the special characters that I use the most. Hopefully other people will find these lists equally useful.
